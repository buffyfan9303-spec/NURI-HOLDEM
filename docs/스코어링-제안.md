# NURI HOLDEM 스코어링 & 로열티 시스템 — 제안서 (v1)

> 사장님이 던진 점수 아이디어를 **법적 안전 · 상위독점 방지 · 단순 · 비현금**에 맞춰 정리한 실행안입니다.
> 이건 "앱에 박는 것"이 아니라 **사장님이 직접 굴려볼 수 있게** 정리한 스펙입니다. 마지막에 구글시트 연동법 포함.

---

## 0. 한 줄 요약 (TL;DR)

- **병렬 리더보드 4종 + 티어 리그 1종**을 월간으로 돌린다. (한 사람이 다 먹지 못하게 — 서로 다른 행동을 보상)
- 점수는 **놀이 성취 지표**로만. **순수익(머니인−바이인) 같은 금전 지표는 공개 점수에서 제외** (법적·고액 유도 위험).
- 보상은 **전부 비현금** — 권력 / 명예 / 편의 / 적립(매장이용권). 현금·환금성 절대 금지.
- 플레이어가 **보는 화면은 딱 3개**: ① 내 리그·순위 ② 내 포인트 ③ 다음 보상까지 진행바. 나머지는 백그라운드 계산.
- 지금 당장은 **CSV 내보내기(이미 구현됨) → 구글시트 수식**으로 점수를 튜닝하고, 룰이 확정되면 앱에 내장.

## 1. 설계 원칙 (사장님 4대 주의사항 → 장치로 변환)

| 사장님 주의 | 구현 장치 |
|---|---|
| **고액 X** | 점수에 금액(원) 미사용. "횟수·참여·스킬"만. 일일 상한으로 큰손 1인 독식 차단 |
| **과도한 현금 X** | 보상 100% 비현금. 할인은 "매장이용권(놀이 입장권)" 형태로 지급 → 환금 불가 |
| **상위 독점 X** | ① 월간 리셋 ② 티어별 분리 리그(실버는 실버끼리) ③ 전월 1~3등 핸디캡 ④ 참여형 추첨(누구나 굿즈 당첨) |
| **복잡 X** | 플레이어엔 3요소만 노출. 점수 통화는 "누리 포인트" 1종으로 통일 |

> 업계 정설과 일치: 글로벌 단일 리더보드는 소수 고인물이 독식해 **대다수를 이탈**시킨다 → 해법은 **주/월 리셋 · 비슷한 패턴끼리 묶는 세그먼트 리더보드 · 달성 가능한 로컬 순위**. 그리고 **단순함이 가입의 핵심**(복잡 = 이탈). (출처 하단)

## 2. 기존 점수제 벤치마크 (사장님 "찾아볼것" 요청)

| 사례 | 핵심 메커니즘 | NURI에 차용 |
|---|---|---|
| **GGPoker Fish Buffet / ChampionPoker** | 레이크·참가 기여 → 포인트 → **티어 승급**, 티어별 적립률↑ | 등급별 5~10% 적립 (사장님 안과 동일) |
| **WSOP Player of the Year / GPI** | 대회 성적(필드 크기·순위) → POY 포인트, **연간 누적** | 명예의전당(올타임, 보상X·명예만) |
| **롤(LoL) 랭크 (아이언→챌린저)** | 같은 티어끼리 경쟁, 시즌 리셋 | 사장님 마스터/챌린저/다이아… 티어가 **이미 이 구조** ✅ |
| **항공 마일리지 / 스벅 별** | 적립 통화 + 등급 혜택, 진행바로 동기부여 | 누리 포인트 1종 지갑 + 진행바 |
| **카지노 컴프(Comp)** | 비현금 혜택(식음료·룸·우선석)으로 충성 유도 | VIP테이블·락커·자켓 = 컴프형 비현금 보상 |

**시사점:** 사장님 아이디어(티어 리그 + 등급 적립 + 비현금 보상)는 이미 업계 정석과 같은 뼈대. 다듬을 곳은 **① 금전 지표 제거 ② 상위독점 방지 장치 ③ 노출 단순화** 세 가지뿐.

## 3. 점수 체계 — 정리본

### 3-1. 헤드라인 리더보드 4종 (월간, 병렬 — 서로 다른 사람이 1등)

| # | 보드 | 측정 | 점수 | 리셋 | 핵심 |
|---|---|---|---|---|---|
| A | **참여왕** (구 바인킹) | 매일 참가자 탑5 누적 | 1등100·2등30·3등10·4등5·5등2 | 월 | **일일 상한** — 하루 최대 1회만 보드 반영 → 큰손 1인 도배 방지 |
| B | **스킬왕** (구 승률왕) | **ITM율**(입상 비율) 또는 회수율 — ⚠️순수익 아님 | ITM 1회당 +점, 표본 5회↑부터 랭크 | 월 | 금액 미사용 → 법적·고액 안전 |
| C | **출석왕** | 방문 횟수 | 1~10회=각 20 · 11~20회=각 10 (체감) | 월 | 꾸준함 보상, 후반 체감으로 독주 억제 |
| D | **뉴페이스** | 가입/첫방문 30일 내 신규만 | 1등30·2등20·3등10 | 월 | 신규 진입장벽↓ (사장님 안 그대로) |

> **B(스킬왕) 변경 강력 권고:** 원안 "머니인−바이인=점수"는 곧 **순수익(현금 손익) 추적**이라 ① 사행성·법적 리스크 ② 고액 플레이어 유리(고액X 위반). → **ITM율(들어간 대회 중 입상 %)** 또는 **칩 회수율(ICM 환산, 원 단위 아님)**, 표본 하한(예: 월 5회 이상)으로 대체. 실력은 보상하되 돈은 안 드러냄.

### 3-2. 티어 리그 (시즌전 — 같은 급끼리 경쟁 = 상위독점 방지의 핵심)

배정 기준 = **월간 참가(머니인) 횟수**. 각 티어가 **자기들끼리** 시상 → 실버가 마스터에게 안 깔림.

| 티어 | 월 참가 | 시상 (1/2/3등) |
|---|---|---|
| 마스터 | 20회+ | 40 / 20 / 10 |
| 챌린저 | 15–19 | 30 / 15 / 10 |
| 다이아 | 10–14 | 20 / 10 / 5 |
| 플래티넘 | 7–9 | 15 / 10 / 5 |
| 골드 | 5–6 | 10 / 5 |
| 실버 | 3–4 | 5 |
| 브론즈 | 1–2 | 참가 도장 + 추첨권만 |

### 3-3. 상위독점 방지 4종 장치 (사장님 "상위독점X" 직결)

1. **월간 리셋** — 모든 보드 매월 0으로. 올타임은 *명예의전당*(보상 없음, 명예만).
2. **티어 분리 리그** — 위 3-2. 비슷한 활동량끼리 경쟁(업계 best practice).
3. **전월 챔피언 핸디캡** — 전월 각 보드 1~3등은 신월 *뉴페이스/언더독·럭키 추첨 제외* 또는 +20% 문턱.
4. **참여형 추첨(가장 강력)** — 방문·참가 1회 = **추첨권 1장** → 월말 굿즈 추첨. **실력 무관 누구나 당첨** → 비현금 + 참여보상 + 반독점 동시 충족.

## 4. 보상 카탈로그 — 전부 비현금 (사장님 안 정리)

| 분류 | 보상 | 비고 |
|---|---|---|
| **권력(Privilege)** | VIP테이블 우선배정 · 자동예약권 · 챔피언전 참가권 · 레지 우선 | 등급/시즌 상위 |
| **명예(Status)** | 닉네임 **LED 전광판** · **챔피언 자켓** · 명예의전당 · 프로필 뱃지/진열장(앱 구현됨) | 바이럴 효과 큼 |
| **편의(Convenience)** | **개인 락커 100개(25×25cm)** · 전용 칩렉/지정석 | 한정 수량 = 희소가치 |
| **적립(Earn)** | **바인 할인권 = 매장이용권 형태** · 등급별 5~10% 적립 · 굿즈샵 교환 | 현금환급 불가로 설계 |

> **포인트 통화 1종 통일:** 리그 시상 + 미션 + 추첨 → **"누리 포인트" 지갑** → 굿즈샵 교환.
> 앱에 **이미 매장이용권 발행/대시보드/실시간 수신 인프라**가 있으니, 포인트도 같은 패턴을 재사용하면 추가 개발 최소화.
> **"95 시스템"(바인 5% 할인 등)**도 현금 차감이 아니라 *할인 이용권 지급*으로 처리 → 환금성 제거.

## 5. 크루(Crew) 시스템

- **레드 / 블랙 / 골드** 3팀. 가입 시 팀 선택(또는 추천인 팀 자동) → **바이럴**.
- 개인의 모든 활동 점수가 **소속 크루 누적**에 합산 → **월 1회 내전(crew전)** → 시즌 크루전.
- 반독점 보너스: 점수가 **팀에 풀링**되므로 스타 몇 명이 전체 보드를 지배하지 못함. 모두가 자기 크루에 기여.
- 보상: 우승 크루 전원 굿즈/포인트 + 크루 전용 뱃지.

## 7. 플레이어가 보는 화면 = 딱 3가지 (복잡 X)

1. **이번 달 내 리그 + 순위** (예: 다이아 3위 / 승급까지 2참가)
2. **내 누리 포인트** (지갑 잔액 + 굿즈샵 바로가기)
3. **다음 보상까지 진행바** (진행 시각화 = 동기부여 정석)

→ 4종 보드·티어·크루·미션 계산은 **전부 백그라운드**. 노출은 위 3개만.

## 8. 📊 구글시트 실시간 공유 — "바로바로 가능한가?" → **가능합니다**

데이터는 Supabase(Postgres)에 있고, 시트로 **준실시간~초단위**까지 흘려보낼 수 있습니다. 쉬운 순서:

| 방법 | 실시간성 | 난이도 | 추천 용도 |
|---|---|---|---|
| **① CSV 내보내기 → 시트 붙여넣기** *(앱에 이미 있음)* | 수동 | ★ 즉시 | **지금 점수 룰 튜닝**용. 세팅 0 |
| **② Apps Script + Supabase REST 폴링** | 1~5분 | ★★ | 읽기전용 자동 갱신 대시보드 |
| **③ Supabase DB Webhook → Apps Script 웹앱(doPost)** | **초 단위** | ★★★ | 순위 바뀌면 시트 자동 반영 |
| **④ Zapier/Make/n8n/Pipedream** | 1~15분 | ★★ (노코드) | 코딩 없이 자동화 |

**가장 추천 경로 (지금 → 나중):**
- **지금(코드 0):** 방법 ① — 장부/통계 CSV 내보내기로 받아서 시트 수식(QUERY/SUMIFS/RANK)으로 4종 보드를 직접 계산·튜닝. 룰이 손에 잡힘.
- **반자동:** 방법 ② — Google Apps Script에서 `UrlFetchApp`로 Supabase **읽기전용 뷰**를 5분마다 당겨와 시트 갱신.
- **완전 실시간:** 방법 ③ — Supabase **Database Webhook**이 행 변경 시 Apps Script **웹앱 URL**로 POST → 시트가 초 단위 반영.

**⚠️ 보안 필수:**
- 시트엔 **`service_role` 키 절대 금지**(유출 시 전 DB 노출). **읽기전용 전용 뷰 + 제한 RLS** 또는 **SECURITY DEFINER 집계 RPC**만 노출.
- 개인정보(실명·전화) 컬럼은 시트로 내보내지 말 것. 닉네임·집계 수치만.
- Sheets는 **자체 웹훅이 없음** → 시트→DB 역방향은 폴링/onEdit뿐(양방향은 충돌처리 복잡). **DB→시트 단방향**을 권장.

## 9. 단계적 실행 로드맵

| 단계 | 내용 | 개발량 |
|---|---|---|
| **Phase 0 (지금)** | CSV(구현됨) → 구글시트 수식으로 점수 룰 확정·튜닝 | 0 |
| **Phase 1** | Apps Script + REST 읽기뷰 폴링 = 준실시간 시트 대시보드 | 소 (앱 변경 0) |
| **Phase 2** | DB Webhook → Apps Script 웹앱 = 초단위 시트 반영 | 소 |
| **Phase 3** | 룰 확정 후 앱 내장: 집계 RPC + 리더보드 탭(현 랭킹 인프라 확장) + 포인트 지갑(이용권 인프라 재사용) | 중 |
